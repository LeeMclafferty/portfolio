Interactive Systems Prototype

This project was made with 3 goals in mind. I wanted to learn how multiplayer replication worked, learn more about dynamic interaction systems, and how to use unreal's UMG system. In this prototype I was able to write a c++ based invnetory and interaction system that is multiplayer compatible and component based so that I am able to attach an inventory or interaction to any Actor in the the world. In addition to that I was able to make a main menu, character creation system and local multiplayer that can connect via himatchi all in the ui.

SeaTek Treasure Recovery

Underwater game design is hard. Not a programming lession, but a lesson I learned making this game. SeaTek Treasure recovery was made in 7 days for an itch.io game jam. It was the first time I was able to due things the right way and have a heavy focus on functionality then assets. Its a mistake I made with my interaction system prototype that I do not want to make again. To the left you will see a video montage of all the steps I took to making this game. The general flow was movment system, pickups, health and damage, enemy movement, enviormental hazards (bubble vents) and finally bringing it all together with art and animations that were all made by me.
